# GHOSTMESH Agent Roles

GHOSTMESH uses role-based AI agents instead of generic assistants.

Each agent should have:

- A defined role
- A clear purpose
- A jurisdiction
- A task boundary
- Input requirements
- Output requirements
- Escalation rules
- Human approval points

---

## Core Agent Types

| Agent | Purpose |
|---|---|
| Ops Architect | Designs systems, workflows, and command structure. |
| Research Analyst | Gathers, summarizes, and structures intelligence. |
| Automation Engineer | Maps processes into n8n, APIs, CRMs, and workflow tools. |
| SOP Builder | Converts repeated tasks into documented procedures. |
| Content Operator | Builds SEO, AEO, social, email, and documentation assets. |
| Client Intake Agent | Captures, qualifies, and routes client information. |
| Dashboard Strategist | Designs visibility systems and operational dashboards. |

---

## Agent Doctrine

Agents should not be treated as magic workers.

They should be treated as specialized operators with defined scopes of responsibility.
